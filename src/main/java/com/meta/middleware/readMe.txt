简介
  中间件术语第一次出现在1968年在德国加尔米施帕腾基兴举办的“NATO软件工程大会”结束后发表的一份报告中。
  这届大会正式确定了“软件工程”的概念，同时还讨论了软件设计、生产和分发等主题。

  1980年代开始流程, 最初作为新应用程序与旧式系统之间的桥梁, 使用中间件集成新老系统

  "中间件"是软件工程学中的重要概念，它的主要作用为：
  开发人员使用中间件为应用程序开发提供支持，并简化设计流程，
  使其将重点工作放在业务逻辑和功能上，而不是不同软件组件之间的连接性(组装)上，子系统/组件之间的连接由中间件完成

关键点
  1.帮助构建应用, 基础能力沉淀
  2.充当桥梁角色, 突出连接系统

例子
  1.数据传输,  MQ-系统间传递数据,  MyBatis-方便CURD
  2.游戏, 游戏引擎-控制音视频

结论
  MyBatis可以叫中间件但是fastjson不算是
  Sentinel可以叫中间件
  RocketMQ可以叫中间件
